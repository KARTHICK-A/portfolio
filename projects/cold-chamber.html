CASE STUDY

Cold Chamber Monitoring System

2025–2026 · Delivered to Delphi TVS
Overview

A 3-node distributed sensor network that monitors cold-chamber temperature profiles from +50 °C down to −20 °C, delivered to Delphi TVS.

The Problem

Cold-chamber testing needs continuous multi-point temperature logging through a 4-hour controlled descent — with no data loss if a link or the PC drops.

System Architecture

Node 1 (ESP32) acquires three temperature channels and transmits every 5 seconds over RS485 half-duplex. Node 2 stores data on SPIFFS flash and relays over LoRa E32. Node 3 bridges LoRa to a PC over USB serial, driving custom Python software. Commands (transmit, store, retrieve, erase) travel PC → Node 3 → Node 2 → Node 1 on an ACK protocol with 3 retries and 3-second timeouts; storage state persists across power cycles.

My Role & What I Built

Full system — firmware for all three nodes, the RS485 and LoRa link design, the storage and command protocol, and the PC-side Python software.

Tech Stack
  • ESP32
  • RS485 half-duplex (DE/RE control)
  • LoRa E32
  • SPIFFS
  • Command/ACK protocol
  • Python
Outcome

Delivered and in use at Delphi TVS.
